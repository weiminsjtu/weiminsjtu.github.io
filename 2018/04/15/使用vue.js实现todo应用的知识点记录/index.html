使用vue.js实现todo应用的知识点记录

  • todo app
  • vue.js

最近刚开始学习vue.js，第一个实践例子便是熟悉的todo应用了。本篇将实现过程中的一些初次接触vue.js的知识点进行记录。

入口文件index.js

在webpack入口文件中，我们实例化一个Vue对象，并传入了options：

new Vue({
    render: (h) => h(App)
}).$mount(root);

这里当new Vue(options)时，会自动运行vm._init方法，后续会跟进vue的源码了解实现原理。
render: (h) => h(App)这里实际上使用了ES6的箭头函数，实际上相当于：

// ES5  
(function (h) {  
  return h(App);  
});  

// ES6  
h => h(App);

而在vue中，这里其实h会传createElement

render: function (createElement) {
return createElement(
  'h' + this.level,   // tag name 标签名称
  this.$slots.default // 子组件中的阵列
)

}

$mount则将vue组件APP挂载到了一个DOM节点root。

app.vue,todo component

<template>
    <div id="app">
        <Header></Header>
        <Todo></Todo>
    </div>
</template>

<script>
    import Header from './todo/header.vue';
    import Todo from './todo/todo.vue';

    export default {
        components: {
            Header,
            Todo
        }
    }
</script>

<style lang="stylus" scoped>
...
</style>

每个vue组件都是由template、script、style三部分组成，分别对应html模版，js模型及业务和css样式。
我们的todo应用由Header和Todo两个组件构成。所引用的其它组件均需要在script里export default {
components:中指出。

style的scoped属性很好的实现了样式私有化的目的，表示该样式只作用于当下的模块。它的渲染规则是：

  1. 给HTML的DOM节点加一个不重复data属性(形如：data-v-2311c06a)来表示他的唯一性
  2. 在每句css选择器的末尾（编译后的生成的css语句）加一个当前组件的data属性选择器（如[data-v-5ef48958]）来私有化样式
app[data-v-5ef48958] {
      ...
}

  1. 如果组件内部包含有其他组件，只会给其他组件的最外层标签加上当前组件的data属性



    …

虽然达到了组件样式模块化的目的，但是会造成一种后果：每个样式的权重加重了。理论上我们要去修改这个样式，需要更高的权重去覆盖这个样式。但是scoped设计的初衷就是让样式变得不可修改。所以我想也没必要去修改样式吧？

todo.vue

我们将其分为三部分，一部分是输入框，一部分是要显示的todo list，以及最下面的数据统计和操作栏。
在最顶层的todo.vue维护着todo list items的数组：

export default {
    data() {
        return {
            todos: [],
            filter: 'all'//默认filter值
        }
    ...
    },
};

todolist，我们定义一个新的组件item.vue:

<template>
    <div :class="['todo-item', todo.completed ? 'completed' : '']">
        <input type="checkbox"
               class="toggle"
               v-model="todo.completed"
        >
        <label></label>
        <button class="destroy" @click="deleteTodo"></button>
    </div>
</template>

<script>
    export default {
        props: {
            todo: {
                type: Object,
                require: true
            }
        },
        methods: {
            deleteTodo() {
                this.$emit('del', this.todo.id)
            }
        }
    }
</script>
<style>...</style>

这里，props用于父组件todo.vue调用子组件item.vue时数据传递，这里接收传过来的todo对象。
在点击每个item的删除按钮时，使用$emit抛出del事件及当前item的id交给父组件进行处理。

在todo.vue如何监听item.vue抛出的事件并进行处理呢？监听了del事件，并用deleteTodo进行处理。

 <Item :todo="todo"
      v-for="todo in filteredTodos"
      :key="todo.id"
      @del="deleteTodo"
>
</Item>

这里的filteredTodos是从todo数组中计算出来的一个数组，进行了状态过滤。v-for遍历该数组并向item组件传递todo对象。

所以归纳来，vue中父子组件之间的通信如下，todo应用剩下的业务逻辑基本也是按照这样来实现的：

  1. 父组件使用 props 把数据传给子组件。
  2. 子组件可以使用 $emit 触发父组件的自定义事件。

总结

本文是对实现todo应用的例子的知识点的简单梳理，但是vue.js的学习还有很多，比如虽然我们使用了v-model进行数据双向绑定，但其背后实现原理还需要进一步探索。还有vue的渲染原理、生命周期、虚拟DOM、vuex、vue-router等，以及webpack的打包原理等等。
