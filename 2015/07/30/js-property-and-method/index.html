傻傻分不清楚之js属性和方法

  • js

js是十分灵活的语言。但也因为其灵活，总让我傻傻分不清一些基本的面向对象的基础。今天一定要好好整理，并攻克之。

例子代码

function Test(name){
        var className = "Test";//私有属性

        this.name = name;//公有属性,在对象实例化后调用

        this.hello = function(){//对象方法
            alert(this.name);
            alert(this.msg());//使用原型方法扩充的方法可以在类内部使用
            alert(this.sex);//使用原型方法扩充的属性可以在类内部使用
            alert(Test.age);//静态属性调用时格式为[对象.静态属性]
        }
    }

     Test.age = 20;//公有静态属性

    Test.Run = function(){//类方法
        alert("我是类方法 Run");
    }

    Test.prototype.sex = "男";//原型属性

    Test.prototype.msg = function(){//原型方法
        alert("我的名字是："+this.name);
    }



    Test.Run(); //类方法也是静态方法，可以直接使用 【对象.静态方法()】
    Test.prototype.msg();//原型方法当成静态方法使用时【对象.prototype.方法()】 
    alert(Test.prototype.sex);//原型属性当作静态属性使用时【对象.prototype.方法()】
    var a = new Test("zhangsan");//对象方法和原型方法需要实例化对象后才可以使用
    a.hello();//对象方法必须实例化对象
    a.msg();//原型方法必须实例化对象
    alert(a.age)://错误，公有静态属性只能使用 【对象.属性】调用

分析

属性

1.私有属性

只能在构造函数内部使用，外面无法访问到，但可以通过闭包进行访问（类似于封装）

2.公有属性

new Test(name)进行对象实例化后，可以访问到name这个公有属性。this.XXX = XXX

3.公有静态属性（类属性）

直接定义在类上，类名.XXX = XXX

4.原型属性

方法

1.对象方法

构造函数中定义的方法，跟公有属性的定义类似，this.XXX = function(){}.对象实例化后可以访问该方法。

构造函数中定义的this.XXX均需要对象实例化后通过 “对象名.XXX” 访问

2.类方法

顾名思义，在类上定义的方法，类名.XXX = function(){}.(实际是静态方法直接调用).不需要生成对象实例。

3.原型方法

类名.prototype.XXX = function(){}

原型方法一般用于对象实例共享，避免了每次调用构造函数时对属性或方法的构造，节省空间,创建对象快.
